🚀 QUICK START (WITH FREE AI)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Get Free Groq API Key (5 minutes, no credit card)
────────────────────────────────────────────────────────
   1. Open: https://console.groq.com/keys
   2. Sign up with email or Google
   3. Copy your API key (starts with gsk_)

STEP 2: Add to .env.local
────────────────────────────────────────────────────────
   Open: .env.local
   Add: GROQ_API_KEY=gsk_YOUR_KEY_HERE
   Save file

STEP 3: Restart Dev Server
────────────────────────────────────────────────────────
   1. Press CTRL+C in terminal (if running)
   2. Run: npm run dev
   3. Wait for: ✓ Ready in XXXms

STEP 4: Test Symptom Checker
────────────────────────────────────────────────────────
   1. Open: http://localhost:3002/symptom-checker
   2. Type a symptom: "fever and cough"
   3. Click: [Analyze Symptoms]
   4. You should see AI-powered analysis!

💡 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

"Analysis Error" on symptom checker?
   → Backend is still working, using mock responses
   → Check server logs in terminal
   → Try with simpler symptom description

Can't access localhost:3002?
   → Check terminal shows "Ready"
   → Try http://127.0.0.1:3002
   → Verify firewall allows port 3002
   → Restart: CTRL+C then npm run dev

Want to use Groq API?
   → Get key at: https://console.groq.com/keys
   → Add to .env.local: GROQ_API_KEY=gsk_YOUR_KEY
   → Restart dev server
   → Test on /symptom-checker
